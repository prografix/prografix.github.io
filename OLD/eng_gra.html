The Graphics and OpenGL.

This page contains two applications written on Visual C++ 6.0 as well as some samples of graphic files.

GraphViewer is an application for viewing graphic files written in a special format (*.gra) for OpenGL.
Version history:
10 Nov 2002 GraphViewer 2.0 - first public version,
22 Mar 2003 GraphViewer 3.0 - added control buttons,
21 Jun 2003 GraphViewer 3.1 - added saving to C++ file.
24 Jan 2004 GraphViewer 3.2 - for dynamic files the opportunity of advance, back and a pause has appeared.
Also you could download an exe-file of the program.

GraphMaker is not a real "maker", but only an application by means of which I created the samples below. The GraphMaker itself doesn’t have any value here, and serves as a model to create gra-files. There is FileWriter class, which performs record files on a disk in a gra-format.

Samples.
Diamond.gra - This sample has been invented by myself.
It is possible to rotate and to move object by mouse ( left and right buttons ) and buttons on toolbar.
BounceBall.gra - This sample has been borrowed from Jan Horn who, in his turn, had reworked the original version belonging to Andreas Gustafsson.
Idea.gra - This is a little bit simplified variant of the "Ideas in motion" program written by Mark J. Kilgard.
Cow.gra - Data for this sample were obtained from a file created by Scott R. Nelson from Sun Microsystems. It is possible to rotate and to move object by mouse and buttons.
PocketWatch.gra - - This sample has been borrowed from Jan Horn too. It is possible to rotate and to move object by mouse and buttons.

Zooming is available for all samples listed above.

The description of a gra-format.

By using conventional notation this format could be represented as the following: 

struct GraFormat
{
    unsigned int flags;
    unsigned int size;
    char comment[size];
    unsigned int version;
    unsigned int data[];
};

flags declares a 4-byte integer intended for 32 one-bit flags. While only the zero bit is involved which informs, that one byte stands for symbol (0) or two bytes are for symbol (1) in the commentary. Other flags must be equal to zero.

size declares a 4-byte integer and is equal to the quantity of bytes in the commentary. If the commentary is absent the integer is equal to 0.

comment declares the commentary itself.

version declares the 4-byte integer equal to the version of GraphicInterface class.

data declares the data which represent coded OpenGL commands and some others.
